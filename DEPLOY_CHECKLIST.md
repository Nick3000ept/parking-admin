# Pre-deploy checklist — СБ3_Паркинг_Админ

Чеклист для прохода **перед каждым** деплоем фронта (`git push` → GitHub Pages) или бэка (`clasp push` + `clasp update-deployment`). Отдельный раздел в конце — для проверок при изменениях в Sheets.

Метки:
- 🤖 **[авто]** — Claude проверяет автоматически через Bash/Grep
- 👤 **[ручное]** — пользователь проверяет в браузере
- 📊 **[Sheets]** — проверка в Google-таблице

---

## A. Рутинный pre-deploy (каждый раз)

### A.1. Код и репозиторий

- [ ] 🤖 **Нет debug-мусора.** Grep по `index.html` и `script.gs` на: `console.log` (если только не оставлен для production-диагностики), `debugger`, `alert(`, `TODO/FIXME`, временные комментарии.
- [ ] 🤖 **`git status` чистый.** В коммит идут только нужные файлы. Нет `*.bak`, `*.tmp`, скриншотов, `.DS_Store`.
- [ ] 🤖 **`GAS_URL` в `index.html`** совпадает с `clasp list-deployments` (production deployment ID `AKfycbxX2sk...iyW6Uhl2`). Если случайно создали New deployment вместо `update-deployment` — URL новый, фронт сломается.
- [ ] 🤖 **Diff осмыслен.** Через `git diff` смотрим что именно меняется в коммите. Не должно быть случайных правок «не туда».
- [ ] 🤖 **Деплой бэка через `clasp update-deployment`**, НЕ `clasp deploy` (это создаёт новый deployment).

### A.2. Smoke-тест (минимальный)

После того как фронт задеплоился (через ~1 мин после `git push`) и/или GAS обновлён:

- [ ] 👤 **Тестовый токен → таблица грузится.** Открыл `https://nick3000ept.github.io/parking-admin/?t=<токен>`. Через спиннер ~3-8с появилась матрица. URL после загрузки очистился от `?t=`.
- [ ] 👤 **Прогресс в шапке корректен.** «ПОМЕЩЕНИЙ N · M · K» и «ПЛОЩАДЬ Σ · Σ_done · Σ_rest м²» — числа адекватные.
- [ ] 👤 **Режим отметки**: клик по кнопке → шапка зеленеет, кнопка темнеет с тенью.
- [ ] 👤 **Один клик ставит отметку.** В режиме отметки кликнул на пустую карточку → она мгновенно зеленеет (optimistic). Открыл Sheets → дата проставилась в нужной строке/колонке.
- [ ] 👤 **Один клик снимает.** Кликнул на зелёную → она вернулась к цвету типа. В Sheets ячейка очистилась.
- [ ] 👤 **Быстрая последовательность set→clear** работает (sequence на стороне фронта защищает от race condition).
- [ ] 👤 **Перезагрузка страницы (F5)** не теряет состояние. Токен из localStorage подхватился, отметки на месте.
- [ ] 👤 **Невалидный токен** → экран «Доступ закрыт», не падает.
- [ ] 👤 **Попап (клик по карточке вне режима отметки)** — открывается с деталями и списком работ. Esc закрывает.
- [ ] 👤 **Ctrl+Click** на 2-3 карточки → внизу синяя панель «Выбрано N · X м²» с тегами. Клик по тегу убирает помещение.

### A.3. Производительность

- [ ] 👤 **doGet первой загрузки укладывается в ~10 сек.** Если дольше — изменилась структура или проблема в GAS.
- [ ] 👤 **doPost (клик в режиме отметки)** — карточка меняется мгновенно (optimistic), сервер отвечает за ~500мс.
- [ ] 👤 **Скролл страницы плавный**, нет лагов при переключении фильтров.

### A.4. Финальный шаг

- [ ] 👤 **Свежий коммит = свежее рабочее состояние.** Если что-то отвалится — `git revert <hash>` или `git reset --hard <prev_hash>`. Точка возврата существует.

## B. При изменении структуры в Sheets

### B.1. Листы

- [ ] 📊 **Лист `Главный`** — все ~1788 помещений на месте, колонки A–L заполнены, колонки работ и `*СП` в правильном порядке (M+).
- [ ] 📊 **Заголовки колонок работ в `Главный`** побайтно совпадают с `Ведомость_работ.Полное название` (опечатки = рассинхрон, работа не появится в UI).
- [ ] 📊 **Формат ячеек дат — `Дата`** (не текст). Тест: вводим `01.01.2026`, должна отобразиться `01.01.2026`.
- [ ] 📊 **Data Validation в `*СП` колонках** — выпадающий список из `Ведомость_подрядчиков!A:A`.
- [ ] 📊 **Лист `Ведомость_работ`** — у каждой работы заполнены 4 колонки (Полное название / Поверхность / Название для площадки / ИД_работы).
- [ ] 📊 **Лист `Ведомость_подрядчиков`** — все активные подрядчики имеют `Статус=Активен` и непустой `Токен`. Запись `Админ` (`ИД_подрядчик=admin`) присутствует если нужны админские отметки. Запись `Наблюдатель` (`ИД_подрядчик=viewer`) — если нужен read-only доступ.
- [ ] 📊 **Лист `Типы_помещений`** — для каждого уникального `Наименование` из реестра есть строка. Через `=COUNTIF('Типы_помещений'!A:A, D2)` — должно быть 1 везде. Если 0 — добавить тип.
- [ ] 📊 **Цвета в `Типы_помещений.Цвет`** — только из пресета (Желтый, Серый, Оранжевый, Красный, Синий, Розовый, Фиолетовый, Голубой, Коричневый). **НЕ Зелёный** — он зарезервирован за «работа готова».

### B.2. GAS Web App

- [ ] 📊 **Deployed как Web App**, `Execute as: Me (kuzkin@acons.group)`, `Who has access: Anyone`.
- [ ] 🤖 **`clasp list-deployments`** показывает production deployment с id `AKfycbxX2sk...iyW6Uhl2`.
- [ ] 🤖 **`appsscript.json`** содержит `webapp.executeAs: USER_DEPLOYING` и `webapp.access: ANYONE_ANONYMOUS`.
- [ ] 🤖 **GAS Daily quota не исчерпана** — у consumer-аккаунтов 90 минут execution time/сутки.

### B.3. Безопасность

- [ ] 👤 **Раздача токенов подрядчикам через защищённый канал** (личный мессенджер). Каждому — `?t=<token>`.
- [ ] 👤 **Админский токен** хранится только у заказчика. Не передаётся через общие чаты/email. При подозрении на утечку — ротация (формула в `Токен` → значения).

### B.4. Бэкап

- [ ] 📊 **Резервная копия Sheets** раз в неделю (`Файл → Создать копию`). Имя с датой.
- [ ] 📊 **История версий Google** — хранит 30 дней, можно откатиться через `Файл → История версий`.

## C. Что НЕ автоматизируем (намеренно)

- **Авто-деплой GAS через GitHub Actions** — `clasp push` локально надёжнее и не требует хранения GAS credentials в CI.
- **Автотесты E2E** — для одностраничника с простыми сценариями smoke-тестов A.2 достаточно.
- **CI/CD pipeline** для Pages — `git push` сам обновляет, дополнительная автоматизация не нужна.

---

## D. Если что-то сломалось после деплоя

1. **Откат фронта**: `git revert <hash>` + `git push`. Через минуту Pages вернётся к предыдущей версии.
2. **Откат GAS**: `clasp list-deployments` покажет версии; `clasp update-deployment --description "rollback" <DEPLOYMENT_ID>` после `clasp pull` старой версии из истории Apps Script Editor.
3. **Откат данных в Sheets**: `Файл → История версий → Просмотреть историю версий` → выбрать состояние до поломки → восстановить.
4. **Полный откат**: восстановить из резервной копии Sheets.
