# Pre-deploy checklist — СБ3_Паркинг_Админ

Чеклист для прохода **перед каждым** деплоем фронта (`git push` → GitHub Pages) или бэка (новая версия GAS Web App). Отдельный раздел в конце — **для первого боевого запуска** (один раз).

Метки:
- 🤖 **[авто]** — Claude проверяет автоматически через Bash/Grep
- 👤 **[ручное]** — пользователь проверяет в браузере
- 📊 **[Sheets]** — проверка в Google-таблице

---

## A. Рутинный pre-deploy (каждый раз)

### A.1. Код и репозиторий

- [ ] 🤖 **Нет утекших секретов в коде.** Grep по `index.html` и `script.gs` на: токены подрядчиков из боевой таблицы, реальный GAS Web App URL (если решили хранить отдельно), Spreadsheet ID. **Если найдено в файле, который попадёт в коммит → стоп, выносим в конфиг.**
- [ ] 🤖 **Нет debug-мусора.** Grep на `console.log` (если только не оставлен намеренно для production-диагностики), `debugger`, `alert(`, `TODO`, `FIXME`, `тест`, `xxx`, временные комментарии.
- [ ] 🤖 **`git status` чистый.** В коммит идут только `index.html`, `script.gs`, `TZ.md`, `README.md`, `DEPLOY_CHECKLIST.md`, `CLAUDE.md`. Нет лишних `*.bak`, `*.tmp`, скриншотов, `.DS_Store`, временных файлов.
- [ ] 🤖 **GAS URL в фронте совпадает с актуальным deployed URL.** Если делали Edit deployment → New version, URL не меняется (см. §7.8 ТЗ). Если случайно создали New deployment → URL новый, фронт надо обновить.
- [ ] 🤖 **Diff осмыслен.** Через `git diff` смотрим что именно меняется в коммите. Не должно быть случайных правок «не туда».

### A.2. Smoke-тест (минимальный)

После того как фронт задеплоился (через ~1 мин после `git push`) и GAS-версия обновлена:

- [ ] 👤 **Свой тестовый токен → таблица грузится.** Открыл `https://nick3000ept.github.io/parking-admin/?t=<тестовый_токен>`. Через спиннер появилась таблица с моими назначениями. URL после загрузки очистился от `?t=` (см. §4 ТЗ).
- [ ] 👤 **Один клик ставит дату.** Кликнул на пустую ячейку → появилась `✓ сегодня`. Открыл Sheets → дата проставилась в нужной строке/колонке. Дата корректная (сегодня по Москве).
- [ ] 👤 **Один клик снимает дату.** Кликнул на отмеченную → confirm → дата исчезла. В Sheets ячейка очистилась.
- [ ] 👤 **Перезагрузка страницы (F5) не теряет состояние.** Токен из localStorage подхватился, таблица перерисовалась.
- [ ] 👤 **Кнопка «Выйти» работает.** Localstorage очистился, появился экран «Доступ закрыт».
- [ ] 👤 **Невалидный токен → отказ.** Открыл с `?t=zzzzz` → экран «Доступ закрыт», не падает.

### A.3. Производительность

- [ ] 👤 **doGet первой загрузки укладывается в ~10 сек.** Если дольше — либо изменилась структура (стало больше колонок), либо проблема в GAS. Не деплоим.
- [ ] 👤 **Скролл таблицы плавный** при моём подрядчике (виртуализация работает).

### A.4. Финальный шаг

- [ ] 👤 **Свежий коммит = свежее рабочее состояние.** Если что-то отвалится — `git revert <hash>` или `git reset --hard <prev_hash>` (если ещё не запушено). Точка возврата существует.

## B. First-launch checklist (один раз перед боевым запуском)

Дополнительно к разделу A:

### B.1. Google Sheets

- [ ] 📊 **Боевая таблица создана** (отдельный файл, не исходный реестр помещений)
- [ ] 📊 **Лист `Главный`** — все ~2000 помещений импортированы, колонки A–L заполнены, колонки работ и `*СП` готовы
- [ ] 📊 **Все назначения проставлены** в колонках `*СП` (применимость работ к помещениям определяется этим)
- [ ] 📊 **Заголовки колонок работ в `Главный`** побайтно совпадают с `Работы.Полное название` (опечатки = рассинхрон)
- [ ] 📊 **Формат ячеек дат — `Дата`** (не текст). Проверить: ввести `01.01.2026` в одну ячейку, должно отобразиться `01.01.2026` (с автодополнением года если ввели `1.1`).
- [ ] 📊 **Data Validation в `*СП` колонках** — выпадающий список из `Подрядчики!A:A`. Попробовать ввести случайное имя → должен заблокировать.
- [ ] 📊 **Условное форматирование работает**:
  - Серый фон в `<работа>` когда `<работа> СП` пуста
  - Красный фон если дата стоит, а СП пуст (нарушение инварианта)
- [ ] 📊 **Лист `Работы` финален** — 8+ работ, у всех заполнены 4 колонки
- [ ] 📊 **Лист `Подрядчики` финален** — все подрядчики со `Статус=Активен` имеют непустой `Токен`, включая запись `Админ` с `ИД_подрядчик=admin`
- [ ] 📊 **Лист `Типы помещений` финален** — для каждого уникального `Наименование` из реестра есть строка. Колонка `Тип_помещения_панель` заполнена везде. Колонка `Цвет` — заполнена у тех типов которые должны быть цветными (остальные = дефолтный серый).
- [ ] 📊 **Все уникальные значения `Главный!D (Наименование)`** есть в `Типы помещений.A`. Через `=COUNTIF('Типы помещений'!A:A, D2)` в служебной колонке — должно быть 1 везде. Если 0 — добавить тип.

### B.2. GAS Web App

- [ ] 📊 **Deployed как Web App**, не «Library» и не «Test deployments only»
- [ ] 📊 **Execute as: Me** (workcacc2025@gmail.com) — иначе нет доступа к Sheets
- [ ] 📊 **Who has access: Anyone** — иначе фронт не сможет дёргать (это публичный endpoint, защита через токен)
- [ ] 📊 **URL deployed Web App записан** в `index.html` (или в config, если вынесли)
- [ ] 🤖 **GAS Daily quota не исчерпана** — у consumer-аккаунтов 90 минут execution time в сутки. На старте проверить что не упёрлись.

### B.3. Безопасность

- [ ] 👤 **Утечка токенов из git history.** Через `git log --all -p | grep -iE "token|spectr|ovchi|alfa"` — не должно быть совпадений в истории. **Если найдено — токены подрядчиков перегенерировать перед раздачей.**
- [ ] 👤 **Боевые токены никогда не были тестовыми.** Если в процессе разработки использовали реальные токены для тестов — перегенерировать через формулу в Подрядчики.
- [ ] 👤 **Раздача токенов подрядчикам через защищённый канал** (личное сообщение в мессенджере, не общий чат, не email-рассылка). Каждому — его ссылка `?t=<token>`.

### B.4. Бэкап

- [ ] 📊 **Резервная копия Sheets** создана сегодня (`Файл → Создать копию`, имя с датой). Сохранить ссылку в надёжном месте.
- [ ] 👤 **README.md в репо** содержит контакты для emergency (кто админ, как переразвернуть, где бэкап)

### B.5. Smoke-тест на реальных подрядчиках

- [ ] 👤 **Выдать ссылку одному подрядчику** (например Овчинникову) → попросить открыть → подтвердить что видит свои назначения и может поставить отметку
- [ ] 📊 **Админ проверил в Sheets** что отметка появилась в правильной ячейке
- [ ] 👤 **Раздать остальным** только после успеха первого

## C. Что НЕ автоматизируем (намеренно)

- **Автоматический deploy GAS** через clasp/CLI — добавляет инфраструктуру и может сломаться. Деплоим GAS вручную через Apps Script UI (см. §7.8 ТЗ).
- **Автотесты E2E** (Playwright/Cypress) — для одностраничника с одним сценарием overhead больше пользы. Smoke-тестов из раздела A.2 достаточно.
- **CI/CD pipeline** через GitHub Actions — `git push` сам обновляет Pages, дополнительная автоматизация не нужна.
- **Уведомления в Telegram о деплое** — не обсуждалось, не добавляем.

---

## D. Если что-то сломалось после деплоя

1. Не паниковать. Откат фронта: `git revert <hash>` + `git push`. Через минуту Pages вернётся к предыдущей версии.
2. Откат GAS: `Deploy → Manage deployments → Edit → Version → выбрать предыдущую версию → Deploy`. URL остаётся тот же.
3. Если данные в Sheets испорчены: `Файл → История версий → Просмотреть историю версий` → выбрать состояние до поломки → восстановить.
4. Если ничего не помогает — восстановить из резервной копии Sheets (см. B.4).
